# Open Source Roadmap

This roadmap highlights areas where contributors can help shape `arch_doc`.

## Good First Issue

- Improve README examples and troubleshooting notes.
- Add more sample architecture configurations.
- Expand remediation text for existing finding codes.
- Add tests for edge-case package layouts.

## Help Wanted

- Dependency graph visualization for larger workspaces.
- Mermaid export improvements and layout options.
- CI integration examples for Windows-based pipelines.
- GitHub Action wrapper for `generate --check` and `validate`.
- Improved Flutter support for plugin and application structures.
- Monorepo improvements for complex package discovery.

## Research Needed

- Advanced architecture rules beyond layer constraints.
- Better diagnostics for generated code and build output.
- More precise contract protocol detection.
- Configuration presets for common workspace styles.
- Optional rendered diagram exports without making generation fragile.
