# Security Policy

## Supported Versions

The current public release line is supported for security reports.

## Reporting A Vulnerability

Please report security concerns privately through the repository security advisory flow when it is enabled. If advisories are not enabled yet, open a minimal issue asking for a private security contact without disclosing exploit details.

Include:

- Affected version or commit.
- Reproduction steps.
- Expected impact.
- Any suggested fix.

`arch_doc` runs locally and does not require network access, external APIs, or generated credentials.
